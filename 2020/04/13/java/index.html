1、面向对象三大特性：封装、继承、多态。

封装是一种对象功能内聚的表现形式，使模块之间耦合度变低，更具有维护性，将类的某些信息隐藏在类内部，不允许外部程序直接访问，而是通过该类提供的方法来实现对隐藏信息的操作和访问；继承使子类能够继承父类，获得父类的部分属性和行为，使模块具有复用性；多态使模块在复用性基础上更加有扩展性，使系统运行更有想象空间，实现方式有覆盖和重载。

2、单例模式及实现？

保证一个类仅有一个实例，并提供一个访问它的全局访问点。
实现方式主要有懒汉式和饿汉式。
1、懒汉式，线程不安全

2、懒汉式，通过加锁实现线程安全

3、饿汉式，线程安全,非懒加载，浪费内存

4、双重锁

5、枚举型，非懒加载、线程安全

3、为什么wait方法在object类中，sleep方法在Thread类中？

JAVA提供的锁是对象级的而不是线程级的，每个对象都有锁，通过线程获得。而wait，notify和notifyAll都是锁级别的操作，所以把他们定义在Object类中。这两者都可以让线程暂停一段时间,本质的区别是一个线程的运行状态控制,一个是线程之间的通讯的问题，sleep控制范围是由当前线程决定,没有释放锁。

4、锁

4.1 乐观锁：认为数据一般情况下不会造成冲突，所以只会在数据进行更新的时候，才会判断期间数据有没有改变，实现通常用”版本号”。乐观锁多用于多读的场景，提高系统吞吐量。使用实例：1、JDK并发包中的原子类使用乐观锁的CAS实现；2、数据库乐观锁、缓存乐观锁；
4.2 悲观锁：对数据修改持悲观态度，在每次数据处理过程中都会对数据上锁，使别的需要该数据的任务处于阻塞，直到它拿到锁。多用于多写的场景，保证数据写操作安全。使用实例：1、java 的synchronized关键字和Lock；2、传统关系型数据库，如行锁，表锁，读锁，写锁等；3、基于zookeeper、Redis 的分布式锁；
4.3 自旋锁：
没有获取到锁的线程的两种处理方式，一种是没有获取到锁的线程就一直循环等待判断该资源是否已经释放锁，这种锁叫做自旋锁，它不用将线程阻塞起来(NON-BLOCKING)；还有一种处理方式就是把自己阻塞起来，等待重新调度请求，这种叫做互斥锁。
自旋锁的定义：当一个线程尝试去获取某一把锁的时候，如果这个锁此时已经被别人获取(占用)，那么此线程就无法获取到这把锁，该线程将会等待，间隔一段时间后会再次尝试获取。这种采用循环加锁 -> 等待的机制被称为自旋锁(spinlock)，因为自旋锁避免了操作系统进程调度和线程切换，所以自旋锁通常适用在时间比较短的情况下。由于这个原因，操作系统的内核经常使用自旋锁。
4.4 互斥锁、共享锁：
互斥锁又叫排它锁，一个锁在某一时刻只能被一个线程占有，其它线程必须等待锁被释放之后才可能获取到锁。如ReentrantLock。
共享锁又称读锁，就是允许多个线程同时获取一个锁，一个锁可以同时被多个线程拥有。比如说ReadWriteLock。

5、停止线程的三种方法

5.1、阻塞中断和非阻塞中断
    interrupt()并不会立即中断，而是设置中断标志为true，提供线程的判断处理，如果当前线程在wait、sleep以及join三个方法引起的阻塞状态下，会重设中断标志为false，并抛出一个InterruptedException；
5.2、stop()，线程对象调用会立即停止，不安全不推荐使用；
5.3、设置标记位停止，但其实也是线程自然终止。
